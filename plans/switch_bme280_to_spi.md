# План перехода BME280 с I2C на SPI

## Цель
Переключить подключение датчика BME280 с интерфейса I2C на SPI.

## Текущая конфигурация
- **I2C пины**: SDA=D2 (GPIO4), SCL=D1 (GPIO5)
- **BME280 адрес**: 0x76 или 0x77
- **Библиотека**: Adafruit_BME280 (поддерживает SPI)

## Необходимые изменения

### 1. Добавление SPI пинов в config.h
Добавить константы для SPI пинов ESP8266 (Wemos D1 Mini):
- MOSI: D7 (GPIO13)
- MISO: D6 (GPIO12)
- SCK: D5 (GPIO14)
- CS: D8 (GPIO15) - Chip Select

### 2. Обновление sensor_manager.h
- Изменить тип объекта BME280 для поддержки SPI
- Добавить конструктор с параметрами SPI пинов
- Удалить зависимость от Wire.h

### 3. Обновление sensor_manager.cpp
- Изменить инициализацию BME280 с I2C на SPI
- Использовать `bme.begin(CS_PIN)` вместо `bme.begin(ADDRESS)`
- Удалить проверку I2C адресов
- Обновить проверку статуса сенсора

### 4. Обновление main.cpp
- Удалить инициализацию I2C (`Utils::initI2C()`)
- Удалить сообщения об I2C в ошибках
- Обновить сообщения об инициализации

### 5. Обновление utils.cpp и utils.h
- Удалить функции `initI2C()` и `checkI2CDevice()`
- Удалить зависимость от Wire.h

### 6. Проверка platformio.ini
- Убедиться, что библиотека Adafruit_BME280 поддерживает SPI
- Добавить зависимость от SPI библиотеки (если требуется)

## Схема подключения BME280 по SPI

```
BME280          ESP8266 (Wemos D1 Mini)
VCC    --------  3.3V
GND    --------  GND
SCK    --------  D5 (GPIO14)
MISO   --------  D6 (GPIO12)
MOSI   --------  D7 (GPIO13)
CS     --------  D8 (GPIO15)
```

## Примечания
- Adafruit_BME280 библиотека поддерживает как I2C, так и SPI
- Для SPI используется метод `begin(CS_PIN)` вместо `begin(ADDRESS)`
- SPI интерфейс быстрее и не требует pull-up резисторов
- Нет конфликтов адресов, как в I2C
